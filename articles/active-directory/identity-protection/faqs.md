# <a name="azure-active-directory-identity-protection-faq"></a>Perguntas frequentes do Azure Active Directory Identity Protection

Este artigo inclui respostas para perguntas frequentes sobre o Azure Active Directory (Azure AD) Identity Protection. Para obter mais informações, consulte [Azure Active Directory Identity Protection](../active-directory-identityprotection.md). 

## <a name="why-do-some-risk-detections-have-closed-system-status"></a>Por que algumas detecções de risco têm o status "fechado (sistema)"?

**R:** Essas detecções de risco foram detectadas pela proteção de identidade e, posteriormente, fechadas porque os eventos não eram mais considerados arriscados. Esses eventos não contam para o nível de risco do usuário. 

---

## <a name="do-i-need-to-be-a-global-admin-to-use-identity-protection-in-the-azure-portal"></a>É necessário ser um administrador global para usar o Identity Protection no portal do Azure?
**R:** Nº Você pode ser um leitor de segurança, um Administrador de Segurança ou um Administrador Global para usar o Identity Protection.

---

## <a name="how-do-i-get-identity-protection"></a>Como posso obter o Identity Protection?

**R:** Consulte [Introdução ao Azure Active Directory Premium](../fundamentals/active-directory-get-started-premium.md) para obter uma resposta para essa pergunta.

---

## <a name="how-can-i-sort-users-in-users-flagged-for-risk"></a>Como é possível classificar usuários em "Usuários sinalizados para riscos"?

**R:** Baixe os usuários sinalizados para relatório de risco clicando em **Baixar**  na parte superior da página **Usuários sinalizados para riscos**. É possível classificar os dados baixados com base em campos disponíveis, incluindo Última atualização (UTC).
