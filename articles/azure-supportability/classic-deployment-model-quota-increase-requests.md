# <a name="classic-deployment-model"></a>Modelo de implantação clássico

Modelo de implantação clássico é a geração mais antiga terra do modo de implantação do Azure impõe um limite de cota de vCPU global para máquinas virtuais e conjuntos de dimensionamento de máquina virtual. Modelo de implantação clássico não é mais recomendado e é agora substituído pelo modelo do Resource Manager. Para saber mais sobre a implantação desses dois modelos e vantagem do Gerenciador de recursos se referem à página de modelo de implantação do Resource Manager. Quando uma nova assinatura é criada, uma cota padrão de vCPUs é atribuída a ele. Sempre que uma nova VM a ser implantado usando o modelo de implantação clássico, a soma do uso de vCPUs novos e existentes em todas as regiões não deve exceder a cota de vCPU aprovada para o modelo de implantação clássico. Saiba mais sobre as cotas na página de limites de serviço e assinatura do Azure.

Você pode solicitar um aumento no limite de vCPUs para o modelo de implantação clássico por meio de ajuda + folha de suporte ou os usos + cota folha no portal.
