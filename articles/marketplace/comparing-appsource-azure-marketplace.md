# <a name="comparing-appsource-and-azure-marketplace"></a>Comparar o AppSource com o Azure Marketplace  
As vitrines atendem a requisitos exclusivos do cliente. As vitrines habilitam o direcionamento por função. É possível oferecer a solução ou serviço corretos com base no cliente.  

## <a name="understanding-the-differences-between-the-storefronts"></a>Compreendendo as diferenças entre as vitrines  
Escolher uma vitrine começa com a identificação do público-alvo para sua oferta. Se você estiver segmentando os dois públicos-alvo, será possível publicar uma vez para vender em ambas as vitrines.  
 
Considere os benefícios adicionais das vitrines.   

### <a name="appsource"></a>AppSource  

| Benefícios | Detalhes | 
|:--- |:--- | 
| ***Flexibilidade de cobrança*** | Fornece uma experiência de avaliação. Não oferece uma opção de publicação habilitados para comércio no momento. É possível usar sua infraestrutura de pedidos e cobrança atual sem investimentos adicionais ou alterações. | 
| ***Facilitação de conexões com outros parceiros*** | Fornecedores de software independentes, integradores de sistemas e fornecedores de serviços gerenciados podem ser vinculados a cenários de implementação específicos. É possível vender para novos clientes de forma colaborativa. | 
| ***Automação*** | Fornecedores de software independentes, integradores de sistemas e fornecedores de serviços gerenciados podem ser vinculados a cenários de implementação específicos. É possível vender para novos clientes de forma colaborativa. |  
| ***Vários tipos de nuvem*** | Atualmente, não é compatível com o Azure Stack, o Azure Governamental ou com nuvens regionais. | 
| ***Apresentação em contexto para clientes*** | Alcance mais clientes por meio da experiência no aplicativo para produtos a Microsoft como Dynamics 365, Power BI e Office 365. |  

### <a name="azure-marketplace"></a>Azure Marketplace  

| Benefícios | Detalhes | 
|:--- |:--- | 
| ***Flexibilidade de cobrança*** | Para VMs, as opções de cobrança Pagamento Conforme o Uso utilizam os EAs (Contratos Enterprise) da Microsoft ou modelos de vendas diretas na Web. As opções de preço incluem uma assinatura de Camada gratuita, em que uma oferta é perpetuamente grátis. As opções de preço também incluem uma assinatura Experimente Agora. Uma assinatura Experimente Agora é gratuita por um período limitado, após o qual a VM é convertida em uma assinatura paga. A ativação do BYOL também é uma opção de compatibilidade.<br />Todos os recursos do Azure provisionados são cobrados diretamente do cliente, para as opções de cobrança e as seguintes ofertas de VM.<ul> <li>Aplicativos do Azure: aplicativo gerenciado</li> <li>Aplicativos do Azure: modelo de solução</li> </ul> | 
| ***Facilitação de conexões com outros parceiros*** | No momento, não é possível vincular um provedor de serviços ou parceiros de entrega à sua oferta. Essa funcionalidade será lançada em 2018. | 
| ***Automação*** | No momento, não é possível vincular um provedor de serviços ou parceiros de entrega à sua oferta. Aproveite a SaaS automatizada com o provisionamento de complementos. Use os aplicativos do Azure: modelo de solução para automatizar cenários de coleta e de implantação de dados baseados em SaaS. |  
| ***Vários tipos de nuvem*** | Publique na nuvem pública e em soluções locais usando os tipos a seguir.<ul> <li>Azure Stack</li> <li>Azure Government</li> <li>Nuvens regionais, incluindo a China e a Alemanha</li> </ul> | 
| ***Apresentação em contexto para clientes*** | Disponibilize sua solução na experiência no portal do Azure para pesquisa contextual usando as máquinas virtuais e os aplicativos do Azure: modelo de solução. |
