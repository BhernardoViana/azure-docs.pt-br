# <a name="power-bi-apps-storefront-details-tab"></a>Guia Detalhes da Vitrine de Aplicativos do Power BI

Use a guia **Detalhes da Vitrine** da página **Nova Oferta** para fornecer informações de marketing, vendas e legais para seus clientes potenciais. Essa guia também especifica como gerenciar clientes potenciais gerados do marketplace. Essa forma longa é dividida em seis seções: **Detalhes da Oferta**, **Detalhes da Listagem**, **Artefatos de Marketing**, **Jurídico**, **Atendimento ao Cliente** e **Gerenciamento de Cliente Potencial**.  Um asterisco (*) acrescentado ao rótulo do campo indica que isso é necessário.


## <a name="offer-details-section"></a>Seção Detalhes da Oferta

Nesta seção, você insere as informações gerais sobre sua Oferta do AppSource.

![Seção Detalhes da Oferta da guia Detalhes da Vitrine](./media/offer-details-section.png)

A tabela a seguir descreve o nome e o objetivo desses campos.

|   Campo               |   DESCRIÇÃO                                                                           |
|-----------------------|-----------------------------------------------------------------------------------------|
| **Resumo da oferta**     | Breve finalidade do aplicativo. Comprimento máximo de 100 caracteres.                             |
| **Descrição da oferta** | Descrição do aplicativo. Comprimento máximo de 3000 caracteres, suporta formatação HTML simples. |
|                       |                                                                                         |


## <a name="listing-details-section"></a>Seção Detalhes da Listagem

Essa segunda seção fornece contexto adicional para seu aplicativo: em quais setores ele normalmente é usado, qual categoria melhor se aplica a ele, produtos compatíveis e termos de pesquisa associados.

![Seção Detalhes da Listagem da guia Detalhes da Vitrine](./media/listing-details-section.png)

 A tabela a seguir descreve o nome e o objetivo desses campos.
 
|   Campo                                  |   DESCRIÇÃO                                                        |
| --------------                           | ---------------------                                                |
| **Indústrias**                           | Selecione o setor ao qual seu aplicativo está mais alinhado. Se o seu aplicativo estiver relacionado a vários setores, você poderá deixar isso em branco.      |
| **Categorias**                           | Selecione as categorias relevantes para seu aplicativo. Selecione um máximo de 3.     |
| **Link de ajuda para seu aplicativo**               | URL para uma página que tem a ajuda online para seu aplicativo           |
| **Produtos com os quais seu aplicativo trabalha (máx. 3)** | Lista os produtos específicos com os quais seu aplicativo funciona. É possível listar um máximo de 3 produtos. Para listar um produto, clique no sinal de mais (ao lado de novo) e um novo campo de texto aberto será criado para você inserir o nome de um produto com o qual seu aplicativo trabalha.      |
| **Pesquisar palavras-chave (máx. de 3)**              | O AppSource permite que o cliente pesquise com base em palavras-chave. Você pode inserir o conjunto de palavras-chave pelas quais o aplicativo será exibido para os clientes. Por exemplo, se o aplicativo é "Meu Aplicativo de Emails", aplicativo de emails, envio de email, email podem ser algumas palavras-chave. Escolha palavras que os usuários provavelmente usarão para pesquisar pelo aplicativo na caixa de pesquisa do AppSource. |
|  |  |


## <a name="marketing-artifacts-section"></a>Seção de artefatos de marketing

Essa terceira seção permite o upload da identidade visual e de materiais de marketing.  Ela é dividida em quatro subseções: **Logotipos**, **Vídeos**, **Documentos** e **Capturas de tela**. Os logotipos e capturas de tela são artefatos de marketing obrigatórios, no entanto, todos são altamente recomendados para o melhor apelo do cliente.

![Seção Artefatos de Marketing da guia Detalhes da Vitrine](./media/marketing-artifacts-section.png)

 
|    Campo                             |    DESCRIÇÃO                                                    |
|   -----------                        |    -------------                                                  |
| *Logos*                              |                                                                   |
| **Logotipo da oferta (formato png, 48x48)**   | Exibido no AppSource na visão geral do aplicativo ou os resultados do aplicativo, quando concluir a pesquisa. Apenas o formato png com uma resolução 48 px\*48 px é compatível.  |
| **Logotipo de oferta (formato png, 216x216)** | Exibido na página de detalhes do seu aplicativo no AppSource.  Apenas o formato png com uma resolução 216 px\*216 px é compatível.  |
| *Vídeos*                             |                                                                   |
| **Nome**                             | Nome ou título do aplicativo                                          |
| **URL**                              | URL do vídeo hospedado no YouTube ou Vimeo                              |
| **Miniatura**                        | Imagem em miniatura do aplicativo.  Apenas o formato png com uma resolução 1280 px\*720 px é compatível.   |
| *Documentos*                          | Opcional, mas no máximo três documentos. Os documentos que você enviar aqui aparecerão no AppSource, em “Saiba mais”.  |
| **Nome**                             | Nome ou o título do documento de suporte                              |
| **Arquivo**                             | O documento carregado deve estar no formato pdf                             |
| *Capturas de tela*                        | Opcional, mas no máximo cinco capturas de tela.                        |
| **Nome**                             | Nome ou título da captura de tela                                       |
| **Imagem**                            | A imagem de captura de tela carregada deve estar no formato png com a resolução de 1280 px\*720 px  | 
|   |   |


### <a name="logo-guidelines"></a>Diretrizes de logotipo

Todos os logotipos enviados para o [Portal do Cloud Partner](https://cloudpartner.azure.com) devem seguir as diretrizes:

- Não use um fundo gradiente em seu logotipo.
- Evite colocar texto, até mesmo sua empresa ou nome de marca, no logotipo. A aparência do seu logotipo deve ser "plana" e deve evitar gradientes.
- Não estique o logotipo.


## <a name="legal-section"></a>Seção legal

Esta quarta seção permite que você forneça os dois documentos legais necessários para cada oferta: Política de Privacidade e Termos de Uso.

![Seção Jurídico da guia Detalhes da Vitrine](./media/legal-section.png)

|   Campo                |   DESCRIÇÃO                           |
|------------------------|--------------------------------------   |
| **URL da política de privacidade** | URL para sua política de privacidade postada       |
| **Termos de uso**       | Política como texto sem formatação ou HTML simples     |
|  |  |


## <a name="customer-support-section"></a>Seção Atendimento ao Cliente

Forneça a **URL de suporte** para sua página de suporte do cliente online.  É melhor quando essa página de suporte online fornece aos clientes várias opções de contato, como telefone, email e chat ao vivo. 


## <a name="lead-management-section"></a>Seção de gerenciamento de leads

A última seção permite coletar clientes potenciais gerados por suas ofertas do AppSource. Ele oferece as seguintes opções de armazenamento (de uma lista suspensa) para essa informação de lead.

|    Campo               |   Destino do cliente potencial                               |
|------------------------|--------------------------------------            |
|  **Nenhum**              | Os clientes potenciais não são coletados (o padrão).  |
| **Blobs do Azure (preterido)** | Um [Blob do Azure](https://docs.microsoft.com/azure/storage/blobs/storage-blobs-overview), especificado por um nome de contêiner e uma cadeia de conexão.  Essa opção está preterida. Em vez disso, use **Tabelas do Azure**.  |
| **Tabela do Azure**        | Uma [Tabela do Azure](https://docs.microsoft.com/azure/cosmos-db/table-storage-overview), especificada por uma cadeia de conexão  |
| **Dynamics CRM Online** | Uma instância do [Microsoft Dynamics 365 Online](https://dynamics.microsoft.com/), especificada por uma URL e credenciais de autenticação |
| **Ponto de extremidade HTTPS**     | O ponto de extremidade HTTPS especificado como um conteúdo JSON   |
| **Marketo**            | Uma instância de [Marketo](https://www.marketo.com/) especificada pela ID do servidor, ID munchkin e ID do formulário.   |
| **Salesforce**         | Um banco de dados do [Salesforce](https://www.salesforce.com/) especificado por um identificador de objeto |
|  |  |

Depois de publicar sua oferta, a conexão do cliente potencial é validada e um cliente potencial de teste é enviado automaticamente para o destino especificado. As informações de cliente potencial devem ser gerenciadas continuamente e essas configurações devem ser prontamente atualizadas para refletir sua arquitetura de gerenciamento de clientes atual.
