# <a name="azure-marketplace-participation-policies"></a>Políticas de participação do Azure Marketplace

As Políticas de Participação do Microsoft Azure Marketplace aplicam-se a todos os editores e ofertas do Microsoft Azure Marketplace. Essas políticas complementam os termos e condições dispostos no Contrato de Editor do Microsoft Marketplace. Para participar do Microsoft Azure Marketplace, os editores devem sempre cumprir as políticas descritas e/ou referenciadas neste documento. Se um editor deixar de cumprir todos os termos e condições a qualquer momento, a Microsoft poderá remover a oferta do editor do Azure Marketplace. Poderemos atualizar este documento periodicamente.

## <a name="base-criteria"></a>Critério base

1. Os softwares e serviços oferecidos no Microsoft Azure Marketplace devem cumprir ao menos um dos seguintes critérios: 

    * *Executar no Microsoft Azure*: A função primária do software ou serviço deverá ser executada no Microsoft Azure.
    * *implantável no Microsoft Azure*: os editores devem descrever nas informações de listagem da oferta sobre como o software ou o serviço é implantado no Microsoft Azure.
    * *Integrar-se a um serviço do Microsoft Azure ou estendê-lo*: Em suas informações de listagem de oferta, os editores devem descrever qual serviço do Azure o software ou o serviço integra ou estende. Como o software ou o serviço integra ou estende o serviço do Azure.

1. Os editores devem estar localizados em um país de origem de venda ao qual o Azure Marketplace dá suporte. ## <a name="publishing-offers"></a>Publicando ofertas

1. Os editores devem publicar pelo menos uma oferta no Microsoft Azure Marketplace dentro de 60 dias após a celebração do Contrato de Editor do Microsoft Marketplace.

1. Os editores devem cumprir os requisitos técnicos do Microsoft Azure Marketplace para integração, como definido nas Diretrizes de Publicação do Marketplace, bem como aqueles que podem vir a ser identificados no Portal de Publicação.

## <a name="offer-listings"></a>Listagens de oferta

1. Os editores devem incluir informações detalhadas da oferta em suas páginas de listagem de oferta, as quais devem ser precisas e atualizadas. Tais informações devem incluir, conforme aplicável:
    * *Descrição da oferta*
        * Descrição mínima da oferta
            * Informações de SDK
            * Proposta de valor
        * Descrição recomendada da oferta
            * Informações detalhadas de SDK
            * Proposta detalhada de valor
            * Recursos: três a cinco afirmações objetivas sobre a oferta
            * Benefícios: três a cinco resultados produzidos pelos recursos da oferta
    * *Modelo de preços*: A oferta deve ser compatível com os [modelos de preços](./marketplace-faq-publisher-guide.md) aos quais o Microsoft Azure Marketplace dá suporte, conforme descrito na seção **Modelos de Preços** deste documento.
    * *Link para os detalhes de suporte ao cliente*: Os editores devem fornecer suporte ao cliente comercialmente razoável para suas ofertas no Microsoft Azure Marketplace.  Eles podem incluí-lo como parte dos valores de usuário associadas à oferta ou como uma oferta de suporte para ser comprado separadamente.
    * *Recursos da oferta*: Os recursos incluem, sem limitação, vídeos de demonstração, capturas de tela, white papers, estudos de caso, testemunhos e documentação técnica detalhada sobre como usar a oferta do editor no Microsoft Azure.
    * *Política de reembolso ao cliente*
    * *Termos de uso*
    * *Política de Privacidade*

1. Os editores não podem redirecionar nem maximizar as vendas de clientes do Azure na sua página de listagem de oferta para softwares ou serviços além do que está disponível no Azure Marketplace. Essa restrição não é aplicável a serviços de suporte que os editores vendem fora do Microsoft Azure Marketplace.

1. Os editores não podem anunciar no Azure Marketplace a disponibilidade de suas ofertas em outras plataformas de nuvem.

1. A Microsoft se reserva o direito de editar e revisar detalhes da página de listagem da oferta para garantir a qualidade. Se a Microsoft fizer alterações em quaisquer detalhes das páginas de listagem, ela informará os editores antes da publicação de suas páginas de listagem de ofertas no Microsoft Azure Marketplace.

## <a name="offering-suspension-and-removal"></a>Suspensão e remoção de oferta

1. A Microsoft se reserva o direito de suspender ou remover uma oferta do Microsoft Azure Marketplace por qualquer motivo. Os motivos pelos quais a Microsoft pode remover uma oferta incluem, entre outros:
    * A oferta não foi provisionada por nenhum cliente por seis meses ou mais;
    * A oferta tem uma alta taxa de cancelamento para SKUs pagos;
    * A oferta recebe comentários negativos de clientes constantemente;
    * A oferta recebe muitos tíquetes de suporte constantemente; ou
    * O editor não está em conformidade com os termos e condições do Contrato de Editor do Microsoft Marketplace, com as Diretrizes de Publicação do Marketplace nem com este documento.

1. Por vários motivos, você pode decidir remover sua oferta do Marketplace. A Remoção de Oferta garante que novos clientes não possam adquirir ou implantar sua oferta, mas não tem impacto sobre clientes existentes. O Encerramento de Oferta é o processo de encerrar o contrato de licenciamento e/ou serviço entre você e seus clientes existentes. As diretrizes e as políticas relacionadas ao término e à remoção da oferta são regidas pelo Contrato de Editor do Microsoft Marketplace (consulte a seção "Termos de pagamento"). É possível solicitar a remoção ou o término ao registrar um tíquete de suporte.
