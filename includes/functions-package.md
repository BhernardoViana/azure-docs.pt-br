Para desenvolvimento local no Azure Functions versão2.x, o pacote é usado para [registrar a extensão de associação](../articles/azure-functions/functions-triggers-bindings.md#local-development-azure-functions-core-tools). Para o desenvolvimento da biblioteca de classes C# no Functions 1.x e 2.x, o suporte para a associação é incluído [instalando o pacote](../articles/azure-functions/functions-triggers-bindings.md#local-c-development-using-visual-studio-or-vs-code).

Observe que o código-fonte segue a numeração de versão do WebJobs SDK: o WebJobs SDK versão 2.x é equivalente ao Azure Functions 1.x, portanto, o código do Functions 1.x está no branch v2.x no repositório. As ramificações mestre e desenv. contêm código para Web Jobs 3.x, que é o mesmo do Functions 2.x.
